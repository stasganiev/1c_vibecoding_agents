## Формат сообщения

```
`<ticket>` `<type>`: `<description>`
```

где `<type>` - категория изменений:
- feat - новая функциональность,
- fix - исправление ошибок,
- docs - изменения в документации (комментарии, описания, README),
- style - форматирование кода,
- refactor - улучшение кода без изменения поведения (рефакторинг),
- test - добавление тестов,
- chore - рутинные задачи (обновление зависимостей и т.д.);

`<ticket>` - номер задачи в формате `FT-xxx`;
`<description>` - краткое описание сути изменений на английском языке.

## Примеры

`FT-123 feat: add user authentication module`
`FT-456 fix: correct calculation of total price in order processing`
`FT-789 docs: update API documentation for new endpoints`
