# Чеклист ревью технической документации 1С

Подгружается навыком `reviewer-analit` на этапе анализа. Используй как опорную сетку проверки — иди по всем блокам, не пропуская.

## Критерии анализа архитектурных решений

### 1. Покрытие требований
- Покрыты ли ВСЕ требования заказчика и бизнеса в предложенной реализации?
- Нет ли требований, оставшихся без архитектурного ответа?

### 2. UX и пользовательский путь
- Соответствует ли описанный UI/UX ожиданиям заказчика?
- Будет ли путь пользователя действительно удобным, а не формально закрывающим требование?

### 3. Соответствие стандартам 1С и БСП
- Не нарушают ли решения стандарты разработки 1С?
- Корректно ли используются паттерны БСП (если заявлено их применение)?
- Нет ли дублирования уже существующих алгоритмов БСП при разработке «нового» функционала?

### 4. Оптимальность использования платформы
- Насколько оптимально задействованы возможности платформы?
- Есть ли риски деградации производительности в многопользовательской среде?

### 5. Существующее vs новое
- Адекватен ли выбор между использованием существующего функционала и разработкой нового?
- Будут ли корректно учтены требования при опоре на существующие механизмы?

### 6. Клиент-сервер
- Корректно ли разнесена клиентская и серверная логика?
- Нет ли бизнес-логики на клиенте? Если есть — это ошибка, отметить явно.

## Шкала ранжирования замечаний

Все замечания ранжируй по критичности, начиная с самых критичных:

| Уровень | Когда применять |
|---------|-----------------|
| 🔴 Блокер | Без исправления документацию нельзя брать в разработку |
| 🟠 Критично | Серьёзный архитектурный риск или непокрытое требование |
| 🟡 Средняя | Заметный недочёт, повышает технический долг |
| 🟢 Низкая | Улучшение, не критичное для старта разработки |
| ⚪ Незначительная | Придирки, вкусовые правки |

## Формат записи замечаний

Каждый пункт начинай с плейсхолдера `[ ]` — аналитик-корректор отметит выполнение после правок:

```
[ ] 🔴 Блокер: <суть замечания и что исправить>
[ ] 🟡 Средняя: <суть замечания и что исправить>
```
